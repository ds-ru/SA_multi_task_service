# SA Multi Task Service

Внутренний мультисервис для автоматизации рабочих процессов, связанных с Telegram Ads, проверкой статистики, обработкой отчетов и работой с данными из разных источников.

Проект предназначен для объединения нескольких небольших инструментов в одном интерфейсе: проверки данных, сверки статистики, генерации отчетов, работы с файлами и вспомогательных операций для аналитики и контроля качества данных.

## Основные задачи

- проверка и сверка статистики по рекламным кабинетам;
- работа с отчетами и выгрузками;
- обработка CSV/XLSX/Parquet-файлов;
- взаимодействие с PostgreSQL и ClickHouse;
- подготовка данных для аналитики и внутренних отчетов;
- автоматизация рутинных операций;
- снижение количества ручных проверок;
- быстрый запуск MVP-инструментов под конкретные бизнес-задачи.

## Возможности сервиса

### Проверка статистики

Сервис позволяет выполнять сверки данных между источниками, проверять расхождения в статистике и быстрее находить проблемные участки в данных.

Примеры проверок:

- расходы;
- показы;
- клики;
- статусы;
- удаленные объявления;
- пропущенные или некорректные записи;
- расхождения между базами и отчетными витринами.

### Работа с файлами

Сервис может использоваться для обработки и конвертации файлов:

- загрузка отчетов;
- чтение CSV/XLSX/Parquet;
- предпросмотр данных;
- фильтрация и базовая проверка содержимого;
- подготовка файлов для дальнейшего анализа.

### Работа с базами данных

Проект рассчитан на работу с аналитическими и операционными хранилищами данных:

- PostgreSQL;
- ClickHouse.

Типовые операции:

- выполнение SQL-запросов;
- получение агрегированной статистики;
- сверка данных между таблицами;
- подготовка отчетных выгрузок;
- контроль качества данных.

### Объектное хранилище

Сервис может использоваться для работы с S3-compatible object storage:

- поиск файлов по ключам;
- скачивание архивов и отчетов;
- конвертация выгрузок;
- подготовка файлов для анализа.

## Пример сценариев использования

1. Проверить статистику по рекламному кабинету за выбранный период.
2. Сравнить данные между источником и аналитической витриной.
3. Найти расхождения по расходам, показам или кликам.
4. Загрузить отчет и привести его к нужному формату.
5. Скачать файл из объектного хранилища.
6. Подготовить выгрузку для дальнейшего анализа.
7. Быстро собрать MVP нового внутреннего инструмента без создания отдельного сервиса.
